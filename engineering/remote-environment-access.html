✓ Prerequisites
Requirements Before Starting

Ensure you have the following before accessing environments:

  • GCP Project Access — Dev or Preprod project access
  • IAM Permissions
    • Kubernetes Engine Developer (or higher)
    • Viewer access to the project at minimum
  • Company Google Account — Added to the organization
  • Lens Desktop (optional but recommended) — https://k8slens.dev/
1 Step 0: Install Required CLI Tools
Skip this step if using Cloud Shell only
0.1 Install Google Cloud SDK (gcloud)

macOS (Homebrew)

brew install --cask google-cloud-sdk
gcloud init

Windows

Install from: https://cloud.google.com/sdk/docs/install

After install:

gcloud init
0.2 Install kubectl

Check if already installed:

kubectl version --client

If missing, install via gcloud:

gcloud components install kubectl
0.3 Install GKE Authentication Plugin (Required)

Install the plugin:

gcloud components install gke-gcloud-auth-plugin

Verify installation:

gke-gcloud-auth-plugin --version

Enable the plugin:

export USE_GKE_GCLOUD_AUTH_PLUGIN=True

Add to your shell config (.bashrc, .zshrc, etc.) for persistence

0.4 Authenticate with Google Cloud

Log in to Google Cloud:

gcloud auth login

Set up application default credentials:

gcloud auth application-default login

Verify authentication:

gcloud projects list
2 Step 1: Access Google Cloud Console
  1. Go to https://console.cloud.google.com
  2. Log in with your company account
  3. Click Cloud Shell button (top right)
  4. Wait for initialization to complete
3 Step 2: Select the Target Environment
Check Active Project
gcloud config list project
Set Project for Dev Environment
gcloud config set project crego-app-dev
Set Project for Preprod Environment
gcloud config set project gold-blueprint-451210-q1
4 Step 3: Get GKE Cluster Credentials
Dev Cluster Credentials
gcloud container clusters get-credentials crego-app-dev-cluster --region asia-south1-a --project crego-app-dev
Preprod Cluster Credentials
gcloud container clusters get-credentials crego-app-preprod --region asia-south1-a --project gold-blueprint-451210-q1
Verify Access

After running the credentials command, verify cluster access:

kubectl get nodes

You should see a list of Kubernetes nodes in the cluster.

5 Step 4: Access Services via Port Forwarding
Port-forwarding works only while the terminal is open. Keep the terminal running to maintain the connection.
PostgreSQL

Start port forwarding:

kubectl port-forward -n preprod svc/postgresql 15432:5432

Connect to database:

psql -h localhost -p 15432 -U <username> -d <database>

Replace <username> and <database> with your credentials

RabbitMQ

Start port forwarding:

kubectl port-forward -n dev svc/rabbitmq 15672:15672 &

Access RabbitMQ Management UI:

  • Cloud Shell: Web Preview → Port 15672
  • Local machine: http://localhost:15672

Credentials:

  • Username: guest
  • Password: guest
Redis

Start port forwarding:

kubectl port-forward -n dev svc/redis 16379:6379 &

Connect to Redis CLI:

redis-cli -h localhost -p 16379
MongoDB

Start port forwarding:

kubectl port-forward -n dev svc/mongodb 28017:27017 &

Connect via mongosh:

mongosh --host localhost --port 28017

Use --username and --authenticationDatabase flags if authentication is enabled

6 Step 5: Access Services Using Web Preview
  1. Ensure port-forward is running (terminal still open)
  2. Click Web Preview button in Cloud Shell
  3. Select Preview on port option
  4. Browser opens the service UI automatically
7 Step 6: Access the Cluster Using Lens
Install Lens

Download and install Lens from: https://k8slens.dev/

Configure kubeconfig

Ensure your kubeconfig is set up by running:

gcloud auth login
gcloud auth application-default login
gcloud container clusters get-credentials <cluster-name> --region <region> --project <project-id>
Connect Cluster in Lens
  1. Open Lens application
  2. Go to Catalog → Clusters
  3. Select the cluster you configured
  4. Click Connect
What You Can Do in Lens
  • Browse pods, deployments, and services
  • View logs and events
  • Execute commands inside containers (exec)
  • Monitor resource usage (CPU, memory)
  • Inspect configuration and secrets
⚙ Cleanup & Troubleshooting
Stop All Port-Forwards

Stop all active kubectl port-forward processes:

pkill -f "kubectl port-forward"

Or for a specific port forward, bring it to foreground and press Ctrl+C.

📋 Summary
  • Cloud Shell — Fastest way to access clusters without local setup
  • Local CLI — Full control and persistent authentication for development workflows
  • kubectl — Primary tool for cluster interaction (pods, services, logs, etc.)
  • Port-forwarding — Access internal services (databases, message queues, caches) from your local machine
  • Lens — Visual Kubernetes management, logs, and debugging interface
